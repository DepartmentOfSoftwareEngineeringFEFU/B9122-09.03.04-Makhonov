# Разработка и реализация программного средства для динамического усложнения игр
Программное средства реализовано на языках C++ и C# в качестве подключаемого плагина для игрового движка Unreal Engine 4.\
\
**ВАЖНО**\
Плагин разрабатывался и тестировался на версии UE 4.27. На других версиях возможны проблемы.

## Инструкция по установке
1. Скачайте все файлы в виде единого архива.
2. Разархивируйте папку по следующему пути: (вам не понадобятся файлы Readme.md и папка Screenshots, их можете удалить)\
`C:\Program Files\Epic Games\UE_4.27\Engine\Plugins\Marketplace`
3. Откройте Unreal Engine и создайте Game проект.
4. В открытом проекте необходимо подключить плагин (в верхнем левом углу):\
`Edit -> Plugins -> Категория (AI) -> DBTPluginTest`
5. После того, как вы нашли нужный плагин, поставьте галочку Enable возле него и перезагрузите проект.
6. Плагин установлен.

## Эксплуатация
Весь плагин разделен на две основные функции:
1. Изменение приоритетов Task Node в структуре Behavior Tree.
2. Изменение отдельных параметров в AIController.

### Изменение приоритетов Task Node в структуре Behavior Tree
После добавления плагина, во всех последующих объектах Behavior Tree некоторые узлы будут модифицированы. Узлы Task Node получат категорию Dynamic Behavior, в которой будет параметр категории способности. Узлы Sequence также получат категорию Dynamic Behavior, в которой будет возможность установить предел использования способности. Когда любая способность игрока будет достигать заданный лимит будет выполнятся переопределение приоритетов. Все вышеперечисленные узлы получат возможность добавления их в систему динамического изменения. Соответственно все новые способности должны быть унаследованы от нового класса DBTAbilityBase, в нем хранится числовая переменная, которая фиксирует количество использований дочерних элементов класса.\
Для того, чтобы эти 2 объекта заработали, они должны находится в комнате и (по правилам) категория способности игрока должна быть противоположной категории способности Task Node в Behavior Tree. Только в этом случае приоритеты узлов изменятся.

### Изменение отдельных параметров в AIController
После добавление плагина, во всех AIController появится категория Dynamic Behavior, в которой можно будет активировать динамическое изменение и временной промежуток, спустя который будет запущено изменение параметров. Важно здесь то, что перменные (изменяемые, с полем MAX) которые будут у AI и Player Controller должны быть одинаковы. В папке проекта, в разделе Config создайте json файл, в котором будут хранится изменяемые переменные и назовите его `GASCharacterMaxValues`.
